Importing data

To import data we need to:

  • Identify the file’s location.

  • Know what function or parsers to use.

For the second step it helps to know the file type and encoding.

File types

  • Files can generally be classified into two categories: text and binary.

Text files

  • common data text file format: .csv
  • script files: .r, .qmd
dir <- system.file(package = "dslabs") 
file_path <- file.path(dir, "extdata/murders.csv") 
file.copy(file_path, "murders.csv") 
[1] TRUE
# file.copy(from, to), returns TRUE/FALSE (success/failure)

Advantages of text files

  • An advantage of text files is that we can easily view its content.

  • Exercise:

    • copy murders.csv into your working directory and examine it with less.
    • Then try the Open file RStudio tool.

Line breaks and delimters

  • Line breaks are used to separate rows

  • a delimiter to separate columns within a row.

  • The most common delimiters are comma (,), semicolon (;), space (), and tab (\t).

  • we need to know what delimiter was used to choose a parser to read a file; The same file can be read by multiple parsers.

  • In some cases, the delimiter can be inferred from file extension: csv, tsv

readLines

readLines("murders.csv", n = 3) 
[1] "state,abb,region,population,total" "Alabama,AL,South,4779736,135"     
[3] "Alaska,AK,West,710231,19"

Binary files

  • Opening image files such as jpg or png in a text editor or using readLines in R will not show comprehensible content: these are binary files.

  • binary files have many formats specific to their data type, are not human readable

  • R’s readBin function can process any binary file

data.table fast Parsers

library(data.table)
x <- fread("murders.csv")
class(x)
[1] "data.table" "data.frame"
  • An R object can have multiple classes. data.table is compatible with data.frame. The object x has both data.frame and data.table behavior.
  • data.table core syntax DT[i,j,by]
    • i: filter rows; j: compute/select columns; by: group
x[region == "South", .(avg_total = mean(total)), by = state] |> head(2)  # `.` alias for list()
      state avg_total
1:  Alabama       135
2: Arkansas        93
  • data.table can modify columns inplace (no copyig) using :=
x[, pop_millions := population/1e6]

scan (lower-level base R)

  • The scan function is the most general parser.

  • It will read in any text file and return a vector so you are on your own coverting it to a data frame.

  • Because it returns a vector, you need to tell it in advance what data type to expect:

scan("murders.csv", what = "c", sep = ",", n = 10) #c: as char. 
 [1] "state"      "abb"        "region"     "population" "total"     
 [6] "Alabama"    "AL"         "South"      "4779736"    "135"       
# read the first 10 items

Encoding: ASCII

https://rafalab.dfci.harvard.edu/dsbook-part-1/wrangling/locales.html

  • Computer translates everything into 0s and 1s.

  • ASCII is an encoding system that assigns specific numbers to characters.

  • Using 7 bits, ASCII can represent \(2^7 = 128\) unique symbols, sufficient for all English keyboard characters.

  • However, many global languages contain characters outside ASCII’s range.

Unicode Transoformation Format (UTF)

  • For instance, the é in “México” isn’t in ASCII’s catalog.

  • To address this, broader encodings emerged.

  • Unicode (all writing systems + symbols) offers variations using 8, 16, or 32 bits, known as UTF-8, UTF-16, and UTF-32.

  • RStudio typically uses UTF-8 as its default.

  • ASCII is a subset of UTF-8, if a file is ASCII-encoded, presuming it’s UTF-8 encoded won’t cause issues.

Other Encodings

  • However, there other encodings, such as ISO-8859-1 (also known as Latin-1) developed for the western European languages (Spanish accents like á, é, ñ), Big5 for Traditional Chinese, and ISO-8859-6 for Arabic.
fn <- "calificaciones.csv" 
file.copy(file.path(system.file("extdata", package = "dslabs"), fn), fn)

readLines(fn, n = 1) #return the first line

guess_encoding

  • The readr parsers permit us to specify an encoding.

  • It also includes a function that tries to guess the encoding:

guess_encoding("murders.csv") 
# A tibble: 1 × 2
  encoding confidence
  <chr>         <dbl>
1 ASCII             1
guess_encoding("calificaciones.csv") 
# A tibble: 3 × 2
  encoding   confidence
  <chr>           <dbl>
1 ISO-8859-1       0.92
2 ISO-8859-2       0.72
3 ISO-8859-9       0.53

Specify encoding

Read csv directly from web

url <- paste0("https://raw.githubusercontent.com/", 
              "rafalab/dslabs/master/inst/extdata/murders.csv") 
x <- read.csv(url) 
x |> head(3)
    state abb region population total
1 Alabama  AL  South    4779736   135
2  Alaska  AK   West     710231    19
3 Arizona  AZ   West    6392017   232
  • paste0(): join strings with no separators

Downloading files

download.file(url, "murders.csv")

  • This will download the file and save it on your system with the name murders.csv.

  • You can use any name here, not necessarily murders.csv.

Warning

The function download.file overwrites existing files without warning.
